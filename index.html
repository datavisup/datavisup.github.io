Présention du cours

Objectifs

  • Maîtriser les outils de visualisation, illustrés avec R (avec ses nombreux packages, ggplot, plotly, dygraphs, treemaps, leaflet etc).
  • Visualiser pour mieux comprendre certaines notions de base en actuariat.
  • Apprendre à communiquer les données de façon pertinente et visuelle (visualisation des données géographiques, interactivité des graphiques, production de dashbord, etc)
  • Développer le sens de la créativité
  • Créer un compte github et publier les projets réalisés

Pré-requis

Il est préférabe d’avoir les quelques pré-requis suivants, néanmions des rappels pourront être faits au début de chaque session.

  • Connaissances de R et RStudio
  • Connaissance du langage markdown
  • Connaissances de base html
  • Bases de données structurées (table, json, xml, etc.)

Outils

Outil de programmation principal : Rstudio Packages R :

  • Manipulation des données: data.table, dplyr, tidyr, stringr, lubridate
  • Grahiques : plotly, ggplot2, ggvis, googleVis
  • Cartographie : Leaflet, sp, maptools, maps, ggmap
  • Séries temporelles : zoo, xts, quantmod, dygraphs
  • Widget html: htmlwidgets
  • Application web : Shiny
  • Webscrapping : XML, rselenium, httr

De nombreux packages sont à découvrir par vous-mêmes.

Évaluation

La note finale sera calculée comme la moyenne des deux notes suivantes:

  • note du cours avec les exercices réalisés (moyenne des 4 séances)
  • note du projet

Chaque dossier doit comporter:

  • les fichiers rmd qui servent à générer les fichiers html
  • les fichiers html générés
  • si plusieurs fichiers html existent (certains graphiques peuvent avoir une taille importante), un fichier index.html doit les regrouper avec des liens.

Conseils

Pour chaque séance, il est conseillé de suivre ces quelques consignes générales :

  • Lire la présentation et les questions de la séance en entier, et garder en tête les objectifs.
  • Commencer à créer un fichier rmd et rappeler les objectifs avec vos propres mots. Ce fichier sera le fichier de travail principal où vous introduirez les questions, les réponses en code, et les interprétations des résultats.
  • Il est tout à fait possible de rajouter des commentaires supplémentaires pour illustrer les objectifs.
  • Le rapport rédigé avec le fichier rmd peut suivre un plan personnalisé.
  • Le rapport final pourra être généré en html et un délégué pourra les regrouper et les publier sur ce site sous l’onglet “projets d’étudiants”.
